# 공통 적용 룰

- **.ai/cursor_agent.rules.md** (PHP 업그레이드 핵심 규칙)
- 실제 scan 시 각 파일의 경로를 확인 후, 대상 파일이 PHP인지 체크  
- PHP 파일만 영향·이슈·변경포인트 대상, 경로 및 도메인 정확 표기  
- 모든 핵심 룰은 cursor_agent.rules.md 우선 적용

# 목적: php 업그레이드 이슈 영향 파일/로직 식별, 실제 수정 금지(가이드만)
- 0단계(Scan) 시 반드시 터미널 명령 우선, 파일 본문 읽기는 Batch 실행 시 최소화
- 각 파일의 경로/도메인/이슈(문법, 함수, deprecated, 버전 등)만 간단 표로 식별

# [보고 요약]

1. 영향 파일(경로, 도메인, 주요 php 이슈, 영향범위) 표로 요약  
	- 경로에 따라 "도메인" 컬럼 추가
	- 각 파일별 php 업그레이드 관련 룰 기준으로 scan

2. 수정 필요 로직 짧게 서술(이유, 단독/묶음)  
	- 파일 경로, 도메인명, 주요 php 이슈(단독/묶음)

3. 1단계 파일 묶음 제안  
	- 한 Batch에 동일 도메인 php 파일 우선 그룹핑

- 구현/해결 방법 언급 금지
- 파일·로직 위치만 최소 서술

(예시)
| 파일 경로       | 도메인 | 이슈          | 영향 |
|----------------|--------|---------------|------|
| a/index.php    | 주식   | 단축 태그 ↓   | 단일 |
| common/util.php| 공통   | 함수 deprecated| 다중 |

[수정 필요 로직]
- a/index.php(주식): php 단축 태그 사용(단독)
- common/util.php(공통): 일부 함수 php8 미지원(묶음)

[1단계 제안]
- a/index.php, a/conf.php 등(주식)
- common/util.php, common/route.php 등(공통)
